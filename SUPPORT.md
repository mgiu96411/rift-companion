# Support

Rift Companion is built and supported by a solo developer. Support is best-effort but genuinely
read — every issue and discussion gets looked at.

## Before you file anything

- **[FAQ](FAQ.md)** — bans, permissions, data sources, requirements.
- **[Install & troubleshooting](docs/install.md)** — "damaged" downloads, panels not showing,
  uninstalling.
- **[Changelog](https://riftcompanion.com/changelog)** — your problem may already be fixed;
  update to the latest version first.

## Channels

| You want to… | Go to |
|---|---|
| Report a bug | [Open an issue](../../issues/new/choose) — the template asks for macOS version, app version, and steps |
| Request a feature | [Open a feature request](../../issues/new/choose) — read-only scope note applies |
| Ask a question, share a setup | [Discussions](../../discussions) |
| Report a security or privacy issue | [Private security advisory](../../security/advisories/new) or **security@riftcompanion.com** — see [SECURITY.md](SECURITY.md) |
| Anything else | **feedback@riftcompanion.com** |

## What to include in a bug report

- macOS version (e.g. macOS 14.5, Apple silicon) and Rift Companion version (menu-bar icon → About).
- Game mode (Summoner's Rift, ARAM, Arena, or champ select only).
- What you expected, what happened instead, and steps to reproduce.
- A screenshot helps enormously for overlay-position or rendering issues.

## Scope note

Rift Companion is read-only by design: official local Riot APIs only, no input automation, no
memory reading. Requests that need automation or hidden game state (e.g. enemy ability/ultimate
cooldowns) are out of scope on purpose — see the
[feature-request template](.github/ISSUE_TEMPLATE/feature_request.yml).
